# Webfolder
 _Менеджер файлов с веб-интерфейсом_

> 📖 Цель: разработать кроссплатформенное приложение, позволяющее пользователю управлять файлами на локальной машине через веб-интерфейс.

#### ⚙️ Основные компоненты:

- 📡 __Бэкенд-сервер__: сервис, запускаемый на целевой машине (где хранятся файлы). Оно поднимает HTTP + WebSocket сервер, работает с файловой системой и отдает данные по API.
- 🖥️ __Фронтенд-клиент__: веб-интерфейс, загружаемый из бэкенда. Взаимодействует с сервером по WebSocket для выполнения файловых операций.

### 🚧 Минимальные требования к интерфейсу:
- Просмотр содержимого папок
- Создание папок и файлов
- Просмотр содержимого файлов
- Удаление папок и файлов
- Копирование, вырезание, вставка файлов
### 🧩 Дополнительный функционал интерфейса:
- Upload/Download файлов (удаленный доступ)
- Drag-and-Drop файлов между папками
- Обновление содержимого папки в реальном времени
- Редактирование текста
- Сортировка содержимого папки
- Отображение метаданных
- Глубокий поиск
- Основная информация о системе (удаленный доступ)
- Мультивыбор
- Перемещение в корзину
- Аутентификация (удаленный доступ)
### 🌟 Сверхфункционал (роскошный максимум):
- Предпросмотр markdown, word, excel и т.п.
- Встроенное ide с форматированием и подсветкой синтаксиса
- LSP для json
- SPA
### 🤖 Требования к бэкенду:
- Сервер HTTP
- Сервер WebSocket
- Реализация API
- Аутентификация (удаленный доступ)
- Кроссплатформенность
- Потоковая передача больших файлов
